# decision-page · 交互决策页

## 何时使用

- 积累了 ≥2 个需要用户拍板的决策，或用户明确要求"做成决策页"。
- 单个简单二选一仍可直接在对话里问（AskUserQuestion）；多项、需要背景比较的决策必须用本 skill。

## 流程

### 1. 放置文件

把 `templates/` 下两个文件复制到项目内合适位置（同一目录）：

- git 仓库项目：建议 `docs/decisions/`（或项目既有约定位置，如 ai-calendar 用 `scripts/decide.py` + `docs/todo/`）。遵循项目 git 规范（分支 + PR）。
- 非仓库场景：任意工作目录即可。

复制后目录内应有：`decide.py`、`decisions.html`；首次保存会自动生成 `decisions-log.md`。

### 2. 填充决策数据

编辑 `decisions.html`：

- 把 `__TITLE__`（两处）替换为项目名。
- 把 `const DECISIONS=[/* __DECISIONS__ */];` 填充为决策数组，每项格式：

```js
{id:"C1",                       // 决策编号，与待办/任务文档对应
 title:"进度反馈做到什么程度",     // 一句话标题
 doc:"03-experience.md",        // 背景文档（可写 "—"）
 ctx:"一两句背景：现状、为什么需要决策、代价对比。",
 opts:[
  {k:"A",t:"选项标题",d:"一句话说明取舍",rec:true},   // rec:true = 推荐项（每题恰好一个）
  {k:"B",t:"另一选项",d:"说明"}]}
```

要求：每项必有推荐（`rec:true`）；选项互斥且穷尽合理路径（2~4 个）；背景里给出代价/收益对比，让用户不读外部文档也能选。

### 3. 告知用户

给用户一条可直接执行的命令（替换实际路径）：

```bash
python3 <目录>/decide.py        # 零依赖，自动打开浏览器；--port N 换端口，--no-browser 不开浏览器
```

并说明：选完点「保存决策」→ 结果写入同目录 `decisions-log.md` → 回到对话说"决策已保存"即可。

### 4. 回填执行（后续轮次）

用户说"决策已保存"（或新会话发现日志有新条目）时：

1. 读 `decisions-log.md` 最新条目（带 `待智能体回填` 注释的）。
2. 把结论回填到项目的决策表/待办文档，解锁对应任务。
3. 把该条目注释改为 `已回填（日期）`。
4. 按结论开始执行（遵守项目工作流，如分支 + PR）。

## 注意

- `decide.py` 只绑定 127.0.0.1，无外部依赖；不要改成监听 0.0.0.0。
- 页面以 `file://` 直接打开时会降级为下载结果文件（无法直写），引导用户用 `decide.py` 启动。
- 决策日志是治理记录：在 git 项目里应当入库。
